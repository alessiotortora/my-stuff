# My Stack

This is what I reach for when I want to build something fast without giving up control. It changes over time, but the current setup is the most fun I've had shipping side projects.

## Frameworks: TanStack Start & Astro

I use [TanStack Start](https://tanstack.com/start) for apps where I need real state, server functions, and a proper router. [Astro](https://astro.build/) is what I pick for content-heavy sites where most of the page should ship as HTML and the interactive bits stay islands.

## API: Hono & oRPC

[Hono](https://hono.dev/) is a tiny, fast web framework that runs anywhere — Workers, Bun, Node — and gets out of the way. On top of it I use [oRPC](https://orpc.unnoq.com/) for end-to-end type safety, so calling a procedure from the client feels like calling a local function.

## Database: PostgreSQL + Drizzle

Postgres is my default. [Drizzle ORM](https://orm.drizzle.team/) makes it pleasant: schemas live in TypeScript, migrations are diff-based, and the types flow straight through the rest of the app.

## Styling: Tailwind & shadcn/ui

[Tailwind](https://tailwindcss.com/) for the utility-first speed; [shadcn/ui](https://ui.shadcn.com/) for accessible primitives I can paste in and reshape. Together they cover almost every UI I need to build, without locking me into someone else's design system.

## Animation: Motion & GSAP

For most interactions, [Motion](https://motion.dev/) is enough — it's React-native, the API is great, and springs are first-class. When I need timeline-based choreography or scroll-driven sequences, I reach for [GSAP](https://gsap.com/) instead.

## AI: Claude Code & Cursor

[Claude Code](https://claude.com/claude-code) in the terminal for bigger refactors and multi-file work; [Cursor](https://cursor.com/) when I want AI inside the editor with quick inline edits. Both feel less like "autocomplete" and more like a collaborator that's read the codebase.
